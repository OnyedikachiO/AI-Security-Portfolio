# AI/ML Security Threats

Understand the foundations of Artificial Intelligence (AI), Machine Learning (ML), Large Language Models (LLMs), and modern AI security threats while exploring how AI is leveraged for both offensive and defensive cybersecurity operations.

---

## Overview

This documentation, explored the core concepts behind Artificial Intelligence, Machine Learning, Deep Learning, Neural Networks, and Large Language Models to build a solid understanding of modern AI systems from a cybersecurity perspective.

I examined how AI technologies introduce new attack surfaces through vulnerabilities such as prompt injection, model theft, data poisoning, privacy leakage, and model drift while also studying how attackers enhance traditional cyber attacks using generative AI.

The project also demonstrated how defenders can leverage AI to improve threat detection, phishing analysis, log investigation, threat hunting, automation, and incident response through practical exercises using an AI security assistant.

### Phase 5 — AI Security Threat Landscape

I analyzed modern AI-specific attack vectors including:

#### Prompt Injection

Studied how attackers manipulate system prompts to override intended model behavior and expose protected information.

#### Data Poisoning

Explored how compromised training datasets can influence model behavior and reduce prediction accuracy.

#### Model Theft

Learned how attackers clone proprietary AI models through repeated API interaction.

#### Privacy Leakage

Examined how AI models may unintentionally reveal confidential information contained within training datasets.

#### Model Drift

Studied how changes in production data gradually reduce model accuracy and create security risks if models are not continuously monitored.

## Key Findings

- AI introduces entirely new attack surfaces that require dedicated security controls.
- Prompt injection remains one of the most significant risks facing LLM-based applications.
- Protecting training data is critical for preventing privacy leakage and data poisoning attacks.
- Continuous monitoring is essential to detect model drift and abnormal AI behavior.
- MITRE ATLAS provides a structured framework for understanding AI-specific adversarial techniques.
- AI significantly enhances defensive security workflows through automation, investigation, and anomaly detection.
- Secure deployment practices are essential to prevent AI systems from becoming organizational liabilities.
